## 30/60/90 framework

### First 30 days — learn
- Set up dev environment (follow setup guide in Notion)
- Pair with your buddy on at least 2 feature tasks
- Read the handbook (this document and linked sections)
- Ship one small PR in week 2 — something real, not a toy task
- 1:1 with Lukas in week 1, week 2, week 4

### Days 31-60 — contribute
- Pick up solo tasks from the sprint board
- Participate in code review as both author and reviewer
- Shadow one incident response if one occurs
- Give feedback on onboarding experience to Lukas by day 45

### Days 61-90 — own
- Lead at least one feature end-to-end
- Present a short demo in team meeting
- Identify one thing to improve in the codebase or process and propose it

## Buddy assignment
Every new hire gets a buddy for the first 30 days. Buddy is a senior or mid engineer, not Lukas. Buddy is responsible for pairing sessions and answering questions.

Buddy assignments are made before the hire's first day — coordinate with Lukas.

## First week checklist
- [ ] Dev environment set up
- [ ] Slack channels joined (list in Notion)
- [ ] Calendar invites accepted (team standup, 1:1 with Lukas, retrospective)
- [ ] Access to Linear, GitHub, AWS console, Datadog provisioned
- [ ] Handbook read (at minimum: team-norms, code-review, incident-response)
- [ ] Buddy meeting scheduled
